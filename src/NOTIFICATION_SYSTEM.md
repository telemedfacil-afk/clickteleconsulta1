# Notification System Documentation

## 1. Database Schema

### `notifications` Table
Stores all notifications generated by the system.
- `id`: UUID (Primary Key)
- `user_id`: UUID (Foreign Key to auth.users) - Recipient of the notification.
- `type`: TEXT - 'message', 'appointment', 'system'.
- `title`: TEXT - Short title (e.g., "Nova Mensagem").
- `message`: TEXT - Main content.
- `details`: TEXT - Optional detailed info (e.g., date/time of appointment).
- `related_id`: UUID - ID of the related resource (conversation_id or appointment_id).
- `created_at`: TIMESTAMPTZ.

### `notification_reads` Table
Tracks which notifications have been read by which user.
- `id`: UUID (Primary Key)
- `notification_id`: UUID (FK to notifications).
- `user_id`: UUID (FK to auth.users).
- `read_at`: TIMESTAMPTZ.
- **Constraint**: Unique combination of (notification_id, user_id).

## 2. Triggers and Automation

### Message Notifications (`create_message_notification`)
- **Trigger**: Fires `AFTER INSERT` on `messages` table.
- **Logic**: 
  - Checks the conversation to identify the recipient (if sender is patient -> recipient is doctor, and vice-versa).
  - Lookups the sender's name from `perfis_usuarios` or `medicos`.
  - Inserts a record into `notifications`.

### Appointment Notifications (`create_appointment_notification`)
- **Trigger**: Fires `AFTER INSERT` on `agendamentos` table.
- **Logic**:
  - Fetches patient and doctor details.
  - Inserts two notifications: one for the patient and one for the doctor.

## 3. Frontend Hook: `useNotifications`
Located in `src/hooks/useNotifications.js`.

### Features
- **Fetching**: Uses a Supabase RPC function `get_unread_notifications` (or logic) to fetch only notifications that do NOT have a corresponding record in `notification_reads`.
- **Real-time**: Subscribes to `INSERT` events on the `notifications` table filtered by the current user's ID.
- **Mark as Read**: 
  - `markAsRead(id)`: Inserts a record into `notification_reads` and optimistically removes the notification from local state.
  - `markAllAsRead()`: Bulk inserts read records for all currently displayed notifications.

## 4. Component Integration
The `NotificationBell` component is the UI entry point.
- **Location**: `src/components/NotificationBell.jsx`
- **Usage**: Integrated into `Header.jsx` (Patient) and `DoctorAreaHeader.jsx` (Doctor).
- **Behavior**:
  - Shows a red badge count for unread items.
  - Clicking opens a dropdown with the list.
  - Clicking a notification marks it as read.
  - "Marcar todas como lidas" button available in the dropdown header.
